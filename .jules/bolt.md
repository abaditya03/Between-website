## 2026-04-14 - React Re-renders in Parallax
**Learning:** The `useScrollParallax` hook in `index.html` was storing the scroll position in a state variable `scrollY`, which updated on every single scroll event. Because this hook was used by the main `App` component, it caused the entire application (including complex `ParticleNetwork` and `CompanionBot` components) to re-render constantly during scrolling.
**Action:** When implementing scroll-based animations or parallax effects in React, never store scroll position in state unless absolutely necessary for rendering other conditional components. Instead, use a `useRef` to target the specific DOM element and update its `style.transform` directly within a `requestAnimationFrame` callback. This entirely bypasses the React render cycle and provides butter-smooth scrolling.
