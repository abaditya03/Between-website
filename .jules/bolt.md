## 2024-05-07 - Avoid Math.sqrt in High Frequency Render Loops
**Learning:** In canvas-based animation loops (like the `requestAnimationFrame` particle logic in `index.html`), computing `Math.sqrt` for distance calculations for every possible particle pair per frame is a massive performance bottleneck. The total number of computations scales with $O(n^2)$.
**Action:** When determining distance thresholds (e.g. `dist < 120`), square the threshold beforehand ($120^2 = 14400$) and check against the squared differences in $x$ and $y$ ($dx^2 + dy^2 < 14400$). Only compute the actual square root (`Math.sqrt`) if the squared threshold check passes and the exact linear distance is strictly required (e.g., for calculating a proportional opacity for lines). Furthermore, modify the inner loop to start at `j = i + 1` to halve operations and prevent self-comparison ($i == j$).
