## 2025-02-23 - High-Frequency Loop Optimizations in Canvas Animations
**Learning:** In the `ParticleNetwork` component (rendered on the static HTML via Babel), iterating through a network of particles using `for (let j = i; j < particles.length; j++)` in the `animate` function (`requestAnimationFrame` loop) is extremely expensive due to redundant reciprocal network comparisons, and excessive calls to `Math.sqrt` for distance calculations that aren't close enough to render lines.
**Action:** When calculating interactions between particles in a network loop, initialize the inner loop at `j = i + 1` to skip self-comparisons and redundant reciprocal pair calculations (reducing iterations by ~50%). Additionally, in high-frequency loops, compute squared distances first (e.g., `distSq < 14400`) to avoid the heavy computational expense of `Math.sqrt()`, calling it only if the squared distance falls within the interaction threshold.
